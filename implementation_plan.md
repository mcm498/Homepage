# 서울시 구청 사이트 소개 및 포털 홈페이지 구축 계획 (PRD)

본 문서는 서울시 25개 자치구청의 공식 웹사이트를 소개하고 편리하게 접근할 수 있도록 돕는 프리미엄 반응형 포털 홈페이지 구축을 위한 제품 요구사항 정의서(PRD) 및 구현 계획안입니다.

---

## 1. 제품 요구사항 정의서 (PRD)

### 1.1. 서비스 개요
* **서비스명:** 서울시 자치구청 포털 (Seoul District Directory)
* **목적:** 서울시 25개 자치구청의 웹사이트 및 주요 민원 창구 링크를 통합 제공하고, 인터랙티브 지도와 최신 웹 트렌드가 결합된 UI를 통해 사용자가 쉽고 유용하게 필요한 행정 정보에 도달하도록 돕습니다.
* **주요 타겟:** 서울시 주민, 타 지역 전입 예정자, 빠른 민원 처리가 필요한 비즈니스 사용자 등.

### 1.2. 핵심 기능 목록 (Core Features)

1. **인터랙티브 지역 카드형 지도 (Visual Cartogram Map)**
   - 서울의 지리적 구조를 직관적으로 형상화한 프리미엄 반응형 격자 지도(Cartogram)를 제공합니다.
   - 각 구청 카드를 호버하거나 탭하면 하이라이트 효과와 함께 간략한 요약 정보(슬로건, 주소)를 노출합니다.
   - 격자 지도는 모바일 기기에서도 손쉽게 탭할 수 있도록 반응형으로 설계됩니다.

2. **구청 상세 프로필 패널 (Profile Panel)**
   - 지도의 특정 구청을 클릭하거나 목록에서 선택하면 상세 정보를 보여주는 세련된 Glassmorphism 패널이 활성화됩니다.
   - **제공 정보:** 구청 국문/영문 이름, 공식 슬로건, 공식 홈페이지 URL, 민원실(민원여권과 등) 다이렉트 링크, 대표 전화번호, 도로명 주소, 관할 행정 구역 요약.
   - **편의 기능:** 전화번호 및 주소 원클릭 복사 단추, 홈페이지 새창 열기.

3. **내 위치 기준 가장 가까운 구청 찾기 (Geolocation Proximity Finder)**
   - HTML5 Geolocation API를 사용하여 사용자의 실시간 좌표를 가져옵니다.
   - 하버사인(Haversine) 공식을 기반으로 서울시 25개 구청 중 가장 가까운 구청을 계산하여 즉시 강조해 주고, 각 구청 카드에 사용자로부터의 거리를 km 단위로 실시간 표시합니다.
   - 위치 권한을 거부했거나 오류가 발생했을 경우에 대한 부드러운 예외 처리(서울시청 중심 안내)를 포함합니다.

4. **실시간 검색 및 스마트 필터링 (Search & Filters)**
   - **실시간 검색:** 초성 검색 지원 및 국문/영문명, 슬로건 키워드 매칭.
   - **권역별 필터:** 동/서/남/북/중부 권역별 구청 필터링.
   - **정렬 옵션:** 이름순(가나다), 가까운 거리순(위치 수집 시), 북마크 등록순.

5. **자주 찾는 구청 북마크 (Favorites & Pins)**
   - 사용자가 자주 방문하는 구청을 즐겨찾기(Pin)할 수 있습니다.
   - 즐겨찾기된 구청은 최상단 전용 섹션에 모아서 배치되어 빠른 접근을 돕습니다.
   - 브라우저의 `localStorage`를 사용하여 페이지를 새로고침하거나 재방문해도 저장 상태가 유지됩니다.

6. **프리미엄 감성 테마 모드 (Theme Toggle)**
   - 시스템 설정을 자동 반영하는 라이트 모드 및 다크 모드.
   - HSL 기반의 다이내믹 컬러 변수를 사용하여 눈의 피로를 덜고 고급스러운 분위기를 연출합니다.

### 1.3. 디자인 요구사항 (Aesthetics & UX)
* **비주얼 컨셉:** 다크 모드를 기본 컨셉으로 하는 고급스럽고 세련된 'Semi-Transparent Glassmorphic' 레이아웃.
* **색상 팔레트:** 深색 계열 백그라운드, Neon-Green/Blue 포인팅 컬러, 그리고 은은한 그라데이션.
* **타이포그래피:** 가독성이 뛰어난 Google Fonts (Outfit, Noto Sans KR) 웹폰트 패밀리 사용.
* **마이크로 인터랙션:** 카드 호버 시 살며시 떠오르는 듯한(Scale & Glow) 3D 느낌의 전환 애니메이션 적용.

## 2. 기술 스택 및 구현 구조

* **Core:** HTML5, CSS3, ES6 Vanilla JavaScript (별도의 번들러 없이 동작하도록 경량 설계하되 성능 극대화)
* **Layout:** CSS Grid + Flexbox (유연한 2단 반응형 레이아웃)
* **Deployment System:** 단일 HTML/CSS/JS 기반으로 빠른 로딩 속도 보장.

### 디렉토리 구조 (Directory Structure)
프로젝트는 `c:\work space\Homepage` 폴더 아래 생성됩니다.
```
c:\work space\Homepage\
├── index.html       # 웹 어플리케이션의 메인 뼈대 및 SEO 마크업
├── index.css        # 디자인 시스템 (Reset, Variables, Dark/Light 테마, 반응형 그리드)
└── app.js           # 25개 구청 데이터셋, Geolocation 계산식, 이벤트 핸들러 및 상태 관리
```

## 4. 사용자 검토 필요 사항

> [!IMPORTANT]
> **실시간 Geolocation 계산 제공 여부**
> 본 포털은 브라우저의 GPS 위치 정보 권한 획득 시 25개 구청 중 가장 인접한 구청을 계산해 줍니다. 
> 혹시 지도 API(예: 카카오맵, 네이버맵 등)를 화면상에 실제로 띄워서 지도 마커로 위치를 보여주고 싶은지, 아니면 자체 제작한 미니멀하고 직관적인 인터랙티브 SVG/CSS Cartogram 지도로 구성하길 원하는지 결정이 필요합니다. 
> *(추천: 단일 페이지 어플리케이션의 디자인 톤앤매너 유지 및 빠른 로딩을 위해 미니멀리즘 격자 지도(Cartogram)가 디자인적으로 더욱 트렌디하며 적합합니다.)*

> [!TIP]
> **데이터 통계 및 비교 기능**
> 서울 각 구의 통계(인구수, 면적 등)나 대표 캐릭터 이미지를 프로필 패널에 추가해 더 완성도 높은 프리미엄 정보를 제공할 수 있습니다.

## 5. 검증 및 확인 계획 (Verification Plan)

### 수동 테스트 시나리오 (Manual Verification)
1. **검색 기능 검증:** 검색창에 "강" 입력 시 강남구, 강동구, 강북구, 강서구가 필터링되는지 확인.
2. **테마 전환 검증:** 다크/라이트 테마 버튼 클릭 시 HSL 변수가 정상 변경되며 전체 UI 색상이 매끄럽게 페이드아웃/인 되는지 확인.
3. **위치 인식 기능 검증:** "내 주변 구청 찾기" 버튼 클릭 후 브라우저 위치 권한 수락 시 사용자와의 거리가 각 구청 카드에 km 단위로 표기되고 정렬되는지 확인.
4. **로컬스토리지 보존 검증:** 임의의 구청 카드에서 별표(즐겨찾기) 아이콘 클릭 후 페이지를 새로고침했을 때 즐겨찾기 섹션이 그대로 유지되는지 확인.
5. **모바일 반응형 검증:** Chrome 개발자 도구를 사용하여 모바일 뷰포트(375px~430px)에서 깨짐 현상이 없는지 확인.
